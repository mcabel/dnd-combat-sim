## DISCOVERIES / PATTERNS FROM THIS SESSION (reuse these)

1. **Four cantrip registries now well-populated.** 12 cantrips in `CANTRIP_EFFECTS` (added Infestation), 1 in `CANTRIP_ATTACK_ADVANTAGE`, 2 in `CANTRIP_SELF_EFFECTS` (added Shillelagh), 3 in `CANTRIP_AOE_EFFECTS` (added Word of Radiance). 2. **Random-direction forced movement pattern (Infestation).** For cantrips that move the target in a RANDOM direction (Infestation's d4 N/S/E/W): use `rollRandomDirection()` to pick the direction, `directionToDelta(dir)` to convert to a grid-square delta, `isDestinationBlocked(from, to, state)` to check if the destination is off-battlefield OR a wall (using `segmentIntersectsAABB` from los.ts for the wall-intersection check). Set `target.pos` DIRECTLY (forced movement — no OAs, no Booming Blade detonation, mirror Thorn Whip / Lightning Lure). The damage is unconditional on save-FAIL; the move is conditional on the destination being unblocked. Exported for testability: `rollRandomDirection`, `directionToDelta`, `isDestinationBlocked`, `applyRandomMove`. Infestation is the FIRST cantrip with random-direction forced movement — future cantrips that move targets randomly (e.g. a hypothetical "wild magic surge" cantrip) reuse this pattern.
3. **Push-away forced movement pattern (Gust — Session 10 will implement).** For cantrips that push the target AWAY from the caster (Gust): use the same forced-movement pattern (direct pos set, no executeMove, no OAs, no Booming Blade) but compute the push direction as AWAY from the caster (target.pos + (target.pos - caster.pos) normalized * pushDist). Distinct from Thorn Whip / Lightning Lure (pull TOWARD caster) and Infestation (RANDOM direction). Reuse Infestation's `isDestinationBlocked` helper for the blocked-destination check.
4. **Caster-centered AoE pattern (Word of Radiance, mirrors Thunderclap / Sword Burst).** The `CANTRIP_AOE_EFFECTS` registry now has 3 cantrips (Thunderclap, Sword Burst, Word of Radiance). The execute handler is near-identical for all of them: find all creatures within range (Euclidean distance — PHB circle, not Chebyshev square), roll the save for each, apply damage on fail / half on success. Caster excluded. Differences: save ability (Thunderclap=CON, Sword Burst=DEX, Word of Radiance=CON), damage type (thunder/force/radiant), components, source. The pattern is now well-established — any future caster-centered AoE cantrip (e.g. a hypothetical "Pulse of Dawn" cantrip) reuses it with different save/damage/components.

In the damage section, check the flag and add the bonus damage (mirror Divine Smite / Hex patterns). Crit doubles the bonus dice (PHB p.196). 11. **`eq()` takes 3 args (label, a, b), `assert()` takes 3 args (label, cond, detail).** Do NOT pass a detail string to `eq()` — it only takes `label, a, b` and the failure detail is auto-generated from JSON.stringify. Use `assert()` if you need a custom detail string on failure. **Session 9 catch:** the Create Bonfire test initially used `assert('label', cond === true, true)` which passed `true` as the detail (a boolean, not a string). The assertion still worked (the cond was the actual check), but the detail message was wrong. Fixed by using `assert('label', cond === true, 'detail string')` or `assert('label', cond, 'detail string')`. The pattern `assert('label', x?.includes('Y') === true, true)` is WRONG — the third arg should be a string detail. Use `assert('label', x?.includes('Y') === true, 'got: ...')` or just `assert('label', !!x?.includes('Y'), 'got: ...')`.
12. **Save-fail log casing (resolveAttack vs cantrip modules):** `resolveAttack`'s save branch (combat.ts) logs save_fail/save_success with LOWERCASE ability name (e.g. "fails DC 30 dex save") because it uses `${action.saveAbility}` (which is lowercase 'dex'/'con'/'str' etc.). Cantrip MODULES that emit their OWN save logs (Sword Burst, Thunderclap, Word of Radiance) use UPPERCASE ability name (e.g. "fails DEX save"). When testing a metadata-only cantrip that goes through resolveAttack's save branch (Create Bonfire, Acid Splash, Poison Spray, etc.), check for LOWERCASE. When testing a cantrip MODULE that emits its own save log (Sword Burst, Thunderclap, Word of Radiance), check for UPPERCASE. To avoid casing bugs, use `.toLowerCase().includes('dex')` for case-insensitive matching.
