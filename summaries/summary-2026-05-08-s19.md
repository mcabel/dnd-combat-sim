# Session Summary — 2026-05-08 (Session 19)

## Completed: Phase 7.6 — Correct Controlled Mount Action Economy (PHB p.198)

### Core Fix: Controlled vs Independent Mount Modes

**Problem:** Previous implementation had combat mounts attacking on their turn even
when "controlled" — this violates PHB p.198 which states controlled mounts can ONLY
Dash, Disengage, or Dodge.

**Solution:** Two distinct modes, default is controlled:

#### Controlled Mount (default, `independentMount: false`)
PHB p.198: "A controlled mount can take only the Dash, Disengage, or Dodge action."
- Mount **cannot attack**
- AI chooses: if enemies adjacent → **Disengage** (rider moves away safely, no OA)
- Otherwise → **Dash** (doubles effective movement pool for rider)
- Rider effectively gets free movement or free Disengage every turn
- Giant Fly: always controlled (defend profile + no attacks)

#### Independent Mount (`independentMount: true`, set via `grantIndependence(mount)`)
- Mount acts on its own initiative slot with full action economy
- Can attack, use any action
- Use for trained war mounts (Warhorse, Giant Eagle, Hippogriff)
- Verified: Warhorse scored 2 attacks in a 3-round fight vs 2 Goblins

### New `src/types/core.ts` field
`independentMount: boolean` — false by default on all Combatants

### New `src/summons/mount.ts` exports
- `setMountMode(mount, independent)` — toggle between modes
- `isIndependentMount(mount)` — true if independent AND being ridden
- `grantIndependence(mount)` — shorthand for setMountMode(mount, true)
- `controlMount(mount)` — shorthand for setMountMode(mount, false)

### `src/engine/combat.ts` rewrite
Mount turn block now:
1. Refreshes movement pool
2. Checks `isIndependentMount()` — if true, falls through to normal turn planning
3. If controlled: checks adjacency → Disengage or Dash, then `continue`

### Test Fix
`concentration_ai.test.ts` factory missing `independentMount: false` — fixed.
