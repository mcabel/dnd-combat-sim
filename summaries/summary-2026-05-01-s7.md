# Session Summary — 2026-05-01 (Session 7)

## Completed This Session

### Phase 4 Tasks 4.1, 4.2, 4.8, 4.9 — Core Mechanics

**Concentration tracking (4.1)** — `src/engine/utils.ts`:
- `startConcentration(caster, spellName)` — sets concentration, overwrites old
- `breakConcentration(caster)` — clears immediately, no save
- `rollConcentrationSave(caster, damageTaken)` — DC = max(10, floor(dmg/2));
  CON mod applied; auto-breaks on failure
- Wired into `resolveAttack()` in `combat.ts`: every time a concentrating creature
  takes damage, a save is rolled. Break logged as `condition_remove` event.
- Going to 0 HP also breaks concentration (in `checkDeath()`).

**Death saving throws (4.9)** — `src/engine/utils.ts`:
- `rollDeathSave(pc)` — nat 20 revives (1 HP), nat 1 = 2 failures, ≥10 = success,
  <10 = failure; 3 successes = stable, 3 failures = dead
- Wired into main combat loop: unconscious PCs roll at start of their turn
- Monsters still die instantly at 0 HP (no saves)
- `deathSaves: { successes, failures } | null` added to `Combatant` type
- Stable = still unconscious but no longer dying (needs healing to wake)

**Sneak Attack (4.2)** — `src/engine/utils.ts`:
- `canSneakAttack(rogue, action, hasAdv, hasDisadv, allyAdjacent)` — checks finesse/ranged
  weapon, advantage OR ally adjacent (not both needed), no disadvantage, once per turn
- `sneakAttackDice(level)` — 1d6 per 2 levels (ceil); level 1 = 1d6
- `usedSneakAttackThisTurn: boolean` added to `Combatant` type; reset each turn
- Wired into `resolveAttack()`: after hit confirmed, checks SA eligibility,
  rolls extra dice, marks flag. SA damage logged separately.
- Finesse weapon detection by name list (rapier, shortsword, dagger, scimitar, etc.)

**Pack Tactics (4.8)** — `src/engine/utils.ts`:
- `hasPackTacticsAdvantage(attacker, target, battlefield)` — checks trait name
  'Pack Tactics', finds non-incapacitated ally within 1 square of target
- Wired into `resolveAttack()`: pack tactics advantage combined with base
  `attackAdvantageState()` before rolling

**Temp HP (supporting 4.x)** — `src/engine/utils.ts`:
- `grantTempHP(target, amount)` — no-stack, takes higher
- `applyDamageWithTempHP(target, amount)` — absorbs temp HP first, overflow to real HP
- All resolveAttack damage paths now use `applyDamageWithTempHP`

**New test file: `src/test/mechanics.test.ts`** — 45 deterministic assertions
covering all 5 mechanics above with edge cases.
