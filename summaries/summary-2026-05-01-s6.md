## Bug Fixed: Dash Loop

**Root cause:** `executeTurnPlan` ran `moveBefore` THEN Dash action. But Dash grants
extra movement by setting `actor.budget.movementFt += actor.speed`. The movement
destination was already computed at plan time with the pre-Dash budget, so
`spendMovement()` silently failed — creature stayed put. Both creatures Dashed at
each other forever without closing range.

**Fix in `src/engine/combat.ts`:** When `plan.action.type === 'dash'`, execute the
Dash action FIRST (doubling the budget), then execute the movement with the doubled
budget. All other actions still use normal order (move → action).

**Result:** Creatures now close the gap in one Dash turn and fight normally.
Fighter vs Larva (200 runs): party 100%, avg 2.45 rounds.
