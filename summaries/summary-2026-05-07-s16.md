**New `src/scenarios/benchmark.ts`** — CLI tool that runs ALL registered presets
with configurable sample size and prints a ranked color-coded comparison table:
```bash
npx ts-node src/scenarios/benchmark.ts --runs 100
```

**Updated `src/summons/registry.ts`** — Mount/familiar role system:
- New `SummonRole` type: `friendly_mount | combat_mount | familiar | hostile_minion | independent`
- New `trueCR` field: effective CR for encounter budget math
  - Defend-only mounts with no attacks (Giant Fly) = trueCR 0
  - A creature that can only Defend loses 100% of solo fights → contributes 0
- New `canAttack: boolean` — false for Giant Fly (no attack in stat block)
- New `canBeMounted: boolean` — true for friendly_mount and combat_mount roles
- New helper functions: `getFriendlyMounts()`, `getFamiliars()`, `getHostileMinions()`,
  `effectiveCRForBudget(entry)` — uses trueCR not estimatedCR

**Design rules encoded in registry (per user spec):**
- Familiar and friendly mount → always party-side
- Hostile minion → enemy-side, can be ridden by enemy non-mount monsters
- Defend-only mounts (no attacks) → trueCR 0
- Combat mounts that can attack → trueCR = estimatedCR
- In a simulation where ONLY defend-profile mounts are spawned → they lose 100%
  (correct: their role is mobility, not combat)
