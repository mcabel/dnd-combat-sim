# Session Summary — 2026-05-09 (Session 15)

## Completed: Phase 7.2 — Concentration AI

### What was done

**`src/types/core.ts`** — Added `requiresConcentration: boolean` to the `Action` interface.

**`src/parser/fivetools.ts`** — Added `detectConcentration(text)` (keyword regex) and
wired it into `parseAction()`. Any monster action whose description contains the word
"concentration" gets `requiresConcentration: true` automatically.

**`src/parser/pc.ts`** — Added `requiresConcentration` to `weaponToAction()` using a
name-based allowlist: bless, shield of faith, entangle, faerie fire, hex, hunter's mark,
hold person, guiding bolt.

**`src/engine/utils.ts`** — Added `requiresConcentration: false` to `unarmedStrikeAction()`
and `improvisedMeleeAction()`.

**`src/ai/actions.ts`** — Two guards added:
1. `bestAttackAction()` candidate filter now excludes actions where
   `a.requiresConcentration && alreadyConcentrating` — prevents silently dropping
   an active concentration spell by casting another.
2. `selectAction()` AoE block: concentration AoE spells are skipped when the caster
   is already concentrating (`wouldDropConcentration` check).

**`src/ai/targeting.ts`** — `smartScore()` already had the concentration-breaking
priority (+70 for castAoEThisCombat proxy). Verified correct — no changes needed.

**`src/test/concentration_ai.test.ts`** — Rewrote broken sections 4–8 that used wrong
factory signatures and undefined variables. Now 33 assertions, all deterministic.

**All other test factories** — Added `requiresConcentration: false` to every `Action`
literal in test files (engine.test.ts, ai.test.ts, combat.test.ts, mechanics.test.ts,
scenario.test.ts, phase4.test.ts).
