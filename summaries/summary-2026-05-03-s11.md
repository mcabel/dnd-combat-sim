### Phase 5.4 — Mount Rules (`src/summons/mount.ts`)
Full PHB p.198 controlled mount implementation:
- `mountCreature(rider, mount)` / `dismountCreature()` — bidirectional link via
  `mountedOn` / `carriedBy` fields on `Combatant`
- `isControlledMount(c)` — true when `carriedBy` is set
- `riderMovementFt()` / `spendMountMovement()` — rider draws from mount's movement pool
- `mountDeathRiderCheck(rider)` — DC 10 DEX save on mount death; fail → prone
- `syncMountInitiative(bf, riderId, mountId)` — mount's initiative slot moved to
  directly follow rider (PHB: "it moves when you do")
- `setupMount(rider, mount, bf)` — convenience: mounts + syncs initiative in one call

**Engine integration (`src/engine/combat.ts`):**
- Controlled mounts skip their own action/bonus action each turn
  (mount still refreshes its movement pool for the rider to use)
- Mount death triggers rider DEX save + auto-dismount + log event

**Type system (`src/types/core.ts`):**
- `mountedOn: string | null` — ID of mount this rider sits on
- `carriedBy: string | null` — ID of rider currently mounted on this creature

### Phase 5.5 — Multi-Encounter Day (`src/scenarios/multiencounter.ts`)
